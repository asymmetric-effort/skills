# Code Humor

## Purpose

Generate jokes rooted in programming culture — the bugs, the tools, the absurdity of software engineering. These should land with developers and make non-developers slightly concerned.

## Prompt

Generate a programming or tech joke. Follow these guidelines:

1. **Know your audience.** The humor comes from shared suffering — broken builds, impossible requirements, legacy code, off-by-one errors.
2. **Real pain is funnier than fake pain.** Reference actual tools, languages, and workflows. "My regex worked on the first try" is funny because it never happens.
3. **Formats that work:**
   - The false confidence: a developer saying something right before disaster
   - The anthropomorphism: treating code/tools as if they have feelings or intentions
   - The euphemism: describing a catastrophe in calm technical language
   - The comparison: "X is like Y, except Y actually works"
4. **Punch up at complexity, not down at people.** Mock the technology, not the person struggling with it.
5. **Callback to universal truths:** naming things is hard, caching is harder, off-by-one errors are everywhere.
6. **Keep it tight.** Setup, punchline. No rambling.

## Examples

- "There are only two hard things in computer science: cache invalidation, naming things, and off-by-one errors."
- "A QA engineer walks into a bar. Orders 1 beer. Orders 0 beers. Orders 99999999 beers. Orders -1 beers. Orders a lizard. Orders NULL beers."
- "'It works on my machine.' — last words before mass deployment."
- "Debugging is like being the detective in a crime movie where you are also the murderer."
- "The best thing about a boolean is that even if you're wrong, you're only off by a bit."
