# Easter Eggs

## Purpose

Design clever hidden features and surprise interactions — the kind that reward curiosity and make someone's day when they stumble onto them.

## Prompt

Design an easter egg or hidden interaction. Follow these guidelines:

1. **Discoverable but not obvious.** The best easter eggs feel earned. They reward exploration, not random clicking.
2. **Contextual humor.** The easter egg should relate to where it lives. A 404 page easter egg should riff on "not found." A settings page one should riff on control.
3. **Non-disruptive.** Easter eggs should delight, not confuse. They must never interfere with real functionality or make users think something is broken.
4. **Formats that work:**
   - Konami code or secret key sequences
   - Hidden responses to specific inputs (try searching for "do a barrel roll")
   - Unusual hover states or long-press behaviors
   - Secret pages or endpoints
   - Console log messages for developers
5. **Reference culture.** Nod to movies, memes, games, or programming lore. The recognition is half the fun.
6. **Make it shareable.** The best easter eggs spread because people want to show others.

## Examples

- Console message: "You're looking at the console? You must be one of us. We're hiring: /careers"
- 404 page: A tiny text adventure game about finding the missing page
- Search bar: Typing "sudo make me a sandwich" returns "ok." instead of search results
- Loading spinner: On the 100th load, it briefly turns into a tiny dancing figure
- Dark mode toggle: The 7th rapid toggle plays a brief disco animation
